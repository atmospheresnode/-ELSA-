ELSA

Introduction.

The Educational Labeling System for Atmospheres (ELSA) is designed to help data providers create PDS4 compliant labels. Many data providers have little or no experience submitting data to the Planetary Data Systems (PDS), regardless whether the standard is PDS3 or PDS4. ELSA's goal is to provide an online environment where data providers can learn about PDS4, build PDS4 bundles, submit completed bundles to Atmos, and manage the reviews/lien resolutions.

In addition to catering to data providers of new data, ELSA would like to cater to those with PDS3 data labels needing to be migrated into PDS4 compliant labels. The Atmos team plans to begin development of the Migrate app within upcoming months. As this idea becomes further progressed, we will release more information about the app.
PDS4 Standard

Introduction to PDS4.

Labels describe data, so labels are metadata. PDS4 is the most current archiving standard for labels. Its predecessor was PDS3. PDS3 described metadata using ODL format. PDS4 describes metadata using XML, eXtensible Markup Language. XML is a more widely available language for metadata management than ODL. In plain English this means that PDS4 labels are written in XML.

ELSA

We like the Top-Down.

At Atmospheres, we chose the Top-Down approach because it gives data providers the big picture of what a Bundle is at the very beginning of their PDS4 journey. The Top-Down approach allows Logical Identifiers (LIDs) to be created in the natural way (Data Provider's Handbook 1.7.0.0, Section 5.2). The PDS4 standard's immense power to archive and organize large data sets revolve around LIDs.

Upon registration with ELSA, the user is asked which agency they work for--NASA, ESA, etc. This creates the agency identifier for each LID throughout the Bundle. When a user begins creating a Bundle, they are asked for the Bundle's name and which Collections their Bundle has. ELSA prevents Bundles of the same name to be created to ensure that Bundles created within ELSA are unique. This provides for an area of improvement within ELSA in which one day we may expand to ensure that the Bundle in creation is unique across all registered products. Until then, data providers should consult their Node to ensure the name is allowed. Once the Bundle name is unique, then regardless of all else that follows, the entire LID is unique.
